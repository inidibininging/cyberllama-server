# cyberllama-client / cyberllama-cserver readme

Frankensteinian proof-of-concept for talking NPCs and simple quest generation.

## Features

- Talk to random NPCs  
- Talk to companions (using AMM)  
- Request simple quests from NPCs (WIP)  
- Generate quest backstories  
- Ask companions what they see  
- NPC mood comments (WIP)
- NPC stats (food, hydration, relationship, fun) (WIP)  
- Comments on district, sub-district, location (WIP) and active quest

## How it works

1. The Cyberllama mod sends the selected dialog option plus world information via HTTPS to a lightweight Python server (cyberllama).
2. Voice lines for V are generated by a TTS function (default: piper via `v_speak`). You can swap this for another engine.
3. The cyberllama server receives dialog from the Lua mod and tracks NPC state: stats, appearance, time of day, location, district/sub-district, combat status, active quest, etc.
4. Dialog generation and mood evaluation:
   - The server queries the configured LLM (e.g., Ollama with Llama 3.1 / 3.2 / LLAVA) to generate dialog options and evaluate mood.
   - Dialog prompts are enriched with external data stored in the wiki folder so NPCs "know" more about quests, locations, brands, etc.
5. Vision:
   - Vision works only if the server runs on the same machine as the game.
   - A screenshot is captured and sent to the LLM for image description to enable comments about what the player sees.
6. Whisper / voice input:
   - Players can talk to NPCs via a keybinding; speech recognition is handled server-side (default used: Whisper).
7. Quest requests:
   - Players can ask NPCs for quests; objective handling is client-side. The server provides voice and dialog.
8. NPC speech engines and voices:
   - Random NPCs use a voice engine by default; each NPC can have a unique voice.
   - Per-NPC voice overrides are available via `npc_overrides` configs (example included).
   - Supported TTS backends included: Piper, Kokoro, Zonos (additional setup required for each).
   - Zonos requires 16-bit 44100 WAV files; its server script loads voices from assets and sends generated audio as bytes to cyberllama.
9. Conversation handling:
   - After NPC speech, the server returns new dialog options for V.
   - Players can record replies (keybinding); recorded audio is processed by the separate whisper service if enabled.
10. Data storage:
   - Dialogs are stored in a SQLite database (character, dialog line, follow-up lines). Current conversation cache is in-memory and reset on mod "reset".

## Installation

Required mods:
- CyberEngineTweaks  
- AppearanceMenuMod  
- CyberScript  
- REDHttpClient  
- REDFileSystem  
- REDData

### Desktop (mod) installation

1. Download this repository.  
2. Place `cyberllama-client` into the mod directory: $CP2077/bin/plugins/cyberllama  
3. Open `init.lua` inside the cyberllama directory.  
4. Set `CyberllamaServer` to your cyberllama server address (default: localhost / 127.0.0.1).  
5. Set `CyberllamaPort` to your cyberllama server port (default: 8081).

### Server notes

- The cyberllama server should run on the same machine as the game for vision features; Zonos-based script can run elsewhere.  
- Zonos server is resource-heavy (≈6 GB VRAM); running everything locally may require ≥12 GB VRAM. Without Zonos, ~8 GB VRAM may suffice.  
- LLMs used are OpenAI-compatible APIs or self-hosted models (Ollama tested). Llama.cpp style setups may produce noisy output without careful tuning.  
- Tested on modest hardware (e.g., GTX 1070 Ti + Intel i7-7700); results will vary by GPU.

## Server requirements

- Windows or Linux  
- Python 3 + pip + venv/conda  
- (Windows) Git Bash for SSL cert generation (or equivalent)  
- PowerShell on Windows recommended  
- Ollama + compatible model that produces stable responses (tested: Llama 3.1 / 3.2 / LLAVA)  
- Kokoro ONNX voice files from Hugging Face: kokoro-v1.0.onnx, voices-v1.0_new.bin  
- Piper (piper-tts) + Piper models  
- Zonos + corresponding ONNX files (optional)

## Server installation

1. Clone this repo.  
2. Run `gen_ssl.sh` (Git Bash on Windows or bash on Linux) to generate SSL certs.  
3. Create and activate a Python virtual environment.  
4. Copy Kokoro ONNX files into the server directory.  
5. Install requirements: `pip install -r requirements.txt`.  
6. Edit `config.yaml` to set file paths and options.  
7. Start the server: `python cyberllama.py`.

If errors occur, you may need to adjust dependency versions.

## Optional: Whisper (voice input) installation

1. Clone the cyberwhisper repository.  
2. Run `gen_ssl.sh`.  
3. Create and activate a Python virtual environment.  
4. Install requirements: `pip install -r requirements.txt`.  
5. Edit `config.yaml` for paths.  
6. Start: `python cyberwhisper.py`.

## Optional: Zonos server installation

Note: Zonos requires 16-bit, 44100 Hz WAV files.

1. Clone the Zonos repository (outside the cyberllama directory).  
2. Follow Zonos install instructions from its repo.  
3. Copy `gen_ssl.sh` and `server.py` from `cyberllama/zonos-script` into the Zonos folder.  
4. Place voice assets into the Zonos `assets` directory.  
5. Run `gen_ssl.sh`.  
6. Start `server.py`.

## Troubleshooting & performance

- Zonos and advanced TTS/LLM setups are GPU/VRAM intensive. Consider separating services across multiple machines.  
- Model choice and prompt engineering significantly affect response quality.
